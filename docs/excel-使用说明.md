# Excel 使用说明

## 目标

这个项目当前使用 `双 Excel` 模式：

- `规则 Excel`
  - 给 `预设页 / 选项页 / 模板页 / 创建页计算` 使用
- `列表 Excel`
  - 给 `列表页 / 创建页添加到列表 / 商品导出` 使用

不要把这两类数据混在一份工作簿里。

## 一、先理解两份 Excel

### 1. 规则 Excel

这份 Excel 是“配置库”。

它负责：

- 国家、平台组合
- 预设参数
- 选项分组
- 模板规则表
- 创建页计算时引用的规则

适合维护：

- 活动费率
- 交易费率
- 提现费率
- 税率
- 贴单费用
- 是否包邮
- 运费表
- 固定附加费表
- 佣金表

### 2. 列表 Excel

这份 Excel 是“商品结果库”。

它负责：

- 创建页添加到列表后的商品记录
- 图片关系
- 变体关系
- 扩展字段
- 结果字段

适合维护：

- 商品名称
- 款号
- 图片
- 变体
- 类目
- 广告类型
- 是否包邮
- 折前价
- 折后价
- 收入
- 利润率
- 净利润
- 各项费用

## 二、规则 Excel 怎么用

### 1. 绑定规则 Excel

在程序里绑定主 Excel 后，会同时服务这些页面：

- 预设
- 选项
- 模板
- 创建

如果你改了 Excel 后想让程序重新读取，重新绑定一次即可。

### 2. 预设页

预设页管理的是：

- 一个组合
- 对应多条预设参数

典型组合：

- 巴西 / 美客多
- 巴西 / TikTok

典型参数：

- 折扣
- 活动费率
- 交易费率
- 提现费率
- 税率
- 贴单费用
- 是否包邮
- 卖家支付运费

建议理解方式：

- 这里放“默认值”和“规则入口”
- 创建页打开后，会先吃这份默认值

相关设计文档：

- [preset-data-format.md](/Users/wx/Documents/learn/linya_admin/docs/preset-data-format.md)
- [page/PresetView.md](/Users/wx/Documents/learn/linya_admin/docs/page/PresetView.md)

### 3. 选项页

选项页管理的是通用下拉项。

目前典型分组：

- 类目
- 广告类型
- 是否包邮

使用原则：

- 如果某个字段是固定可选值，就放到选项页
- 创建页和模板页会复用这些分组

例如：

- `类目` 里放 `默认 / 服饰 / 3C / 家居 / 美妆`
- `广告类型` 里放 `Clássico / Premium`
- `是否包邮` 里放 `是 / 否`

### 4. 模板页

模板页管理的是规则表。

现在建议按“维度 + 结果列 + 记录行”去理解，不要只盯着“一维 / 二维 / 双枚举”这些名字。

一张规则表本质上是：

- 若干个匹配维度
- 若干个结果列
- 多行规则记录

常见例子：

- 固定附加费表
  - 维度：`折后售价`
  - 结果：`固定附加费`
- 运费表
  - 维度：`折后售价`、`重量`
  - 结果：`卖家运费`
- 佣金表
  - 维度：`类目`、`广告类型`
  - 结果：`佣金费率`

字段填写建议：

- 直接填中文字段名
- 例如：`类目`、`广告类型`、`是否包邮`、`折后售价`、`重量`

行为规则：

- 如果模板字段名和选项页分组同名，会自动按下拉理解
- 如果不是选项字段，就按普通输入理解

所以：

- `类目`、`广告类型`、`是否包邮` 这种通常来自选项页
- `折后售价`、`重量`、`交易费率` 这种通常直接录值

### 5. 创建页如何吃规则 Excel

创建页的计算来源分两层：

1. 预设页给默认值
2. 模板页给查表结果

当前典型联动：

- `是否包邮`
  - 预设页给默认值
  - 创建页可以改
- `卖家支付运费`
  - 来自模板规则表查表
- `固定附加费`
  - 来自模板规则表或手工填写
- `广告类型 / 类目`
  - 来自选项页分组

使用顺序建议：

1. 先在预设页确定国家 / 平台组合和默认值
2. 再在选项页维护好所有下拉项
3. 再在模板页把运费表、佣金表、固定附加费表配好
4. 最后在创建页录商品并计算

## 三、列表 Excel 怎么用

### 1. 绑定列表 Excel

列表页使用的是另一份 Excel。

这份 Excel 只负责商品记录，不负责规则。

典型流程：

1. 在创建页录入商品
2. 点击 `添加到列表`
3. 到列表页查看商品
4. 在列表页点击 `保存`
5. 数据同步回列表 Excel

### 2. 列表页写入了什么

列表 Excel 当前会保存这些内容：

- 商品主记录

### 3. 列表 Excel 的几个关键点

#### 绑定后会缓存

列表页绑定的 Excel 路径会缓存。

所以：

- 跳转到别的页面再回来，不应该丢
- 程序重开后，Tauri 模式下也会尝试恢复

#### `添加到列表` 不等于立刻写盘

创建页点了 `添加到列表` 后：

- 先进入程序里的列表数据
- 再由列表页保存到 Excel

所以如果你要落盘，记得去列表页点 `保存`。

#### 列表 Excel 可以反向读回程序

如果你已经有一份列表 Excel，也可以直接在列表页绑定回来。

程序会读取：

- 商品记录
- 图片关系
- 变体关系
- 扩展字段

## 四、图片怎么处理

### 1. 创建页图片

创建页支持：

- 上传图片到程序管理的图片目录
- 也可以添加本地路径 / 链接
- 图片可以预览
- 图片可以和变体关联

### 2. 列表页图片

列表页会优先按商品封面图显示。

如果图片路径有效：

- 程序里可以预览
- 保存到列表 Excel 时，也会尝试写入主表图片位置

### 3. 关于 WPS / Office

当前项目对图片相关的推荐方向，优先还是：

- 程序内维护图片关系
- 列表 Excel 尽量保持可读可同步

## 五、推荐实际使用顺序

建议你平时按这个顺序使用：

1. 准备规则 Excel
2. 绑定规则 Excel
3. 在预设页建组合
4. 在选项页维护下拉项
5. 在模板页维护规则表
6. 到创建页录商品并计算
7. 点击 `添加到列表`
8. 到列表页检查结果
9. 绑定或保存列表 Excel

## 六、常见问题

### 1. 为什么创建页的默认值会变化

因为创建页会吃当前选中预设组合的值。

如果你在预设页改了：

- 是否包邮
- 折扣
- 活动费率
- 交易费率

再回创建页，默认值就可能不同。

### 2. 为什么创建页和选项页里的字段要同名

因为现在很多联动是按“中文字段名”识别的。

例如：

- 模板里填 `类目`
- 选项页里也有 `类目`

程序就能知道：

- 这是一个可选字段
- 应该用对应分组去渲染

所以建议：

- 字段名尽量统一
- 不要今天叫“广告类型”，明天又叫“投放类型”

### 3. 为什么有时候改了 Excel，程序里没立刻变

因为程序有本地缓存，也有绑定状态。

最稳妥的做法：

- 改完 Excel 后重新绑定
- 或者重新进入对应页面确认数据已刷新

### 4. 列表页为什么还要单独保存

因为“添加到列表”和“写回 Excel”是两步：

- 添加到列表：生成商品记录
- 保存：同步到列表 Excel

这样更安全，不会你边输入边覆盖文件。
