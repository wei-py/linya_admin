# 项目功能介绍

## 项目定位

这是一个基于 Excel 驱动的商品利润测算与列表管理工具。

它不是单纯的表单页，也不是单纯的 Excel 编辑器，而是把下面几件事串起来：

- 用 Excel 维护规则和基础配置
- 在程序里完成商品测算
- 把测算结果和商品资料写入列表
- 再同步回另一份商品 Excel

更适合这样理解：

- `规则 Excel` 负责“配置和计算依据”
- `列表 Excel` 负责“商品记录和结果沉淀”

## 核心能力

项目当前主要有 5 个功能模块：

1. 预设管理
2. 选项管理
3. 模板规则管理
4. 创建页商品测算
5. 列表页商品沉淀与 Excel 同步

## 一、预设管理

预设页用于维护：

- 国家
- 平台
- 国家 / 平台组合
- 每个组合下的一组默认参数

典型参数包括：

- 折扣
- 活动费率
- 交易费率
- 提现费率
- 汇损
- 税率
- 贴单费用
- 是否包邮
- 卖家支付运费规则引用

这个模块解决的问题是：

- 不同国家 / 平台的默认参数不同
- 创建页不需要每次从零输入
- 切换组合后，可以直接带出默认计算环境

核心价值：

- 用“组合 + 参数项”方式组织预设
- 支持从 Excel 读取
- 支持修改后同步回 Excel
- 支持在程序内缓存当前预设状态

## 二、选项管理

选项页用于维护可复用的下拉分组。

当前典型分组：

- 类目
- 广告类型
- 是否包邮

这个模块解决的问题是：

- 创建页和模板页不能到处写死下拉项
- 某些字段本质上是“枚举值”，应该集中维护
- 后续新增或修改选项时，不希望改代码

核心价值：

- 统一维护通用选项
- 创建页直接复用
- 模板页按中文字段名自动匹配选项分组
- 选项同样支持 Excel 读写和缓存

## 三、模板规则管理

模板页用于管理规则表。

这个模块不是写死某个平台的某条规则，而是维护“规则表数据”本身。

当前设计已经往通用模型收敛，一张规则表本质上由这些部分组成：

- 基础信息
- 维度配置
- 结果列配置
- 规则记录

### 1. 维度模型

规则表支持按维度匹配结果。

维度可以是：

- 枚举维度
- 区间维度

并且都是可扩展的，不局限于：

- 一维
- 二维
- 双枚举

可以理解成：

- 一张表有若干个匹配维度
- 再根据这些维度命中一条记录
- 最终返回结果列的值

### 2. 模板页能做什么

- 管理规则表列表
- 新建规则表
- 删除规则表
- 编辑规则表名称、类型、来源说明
- 配置维度字段
- 配置结果列
- 维护规则记录
- 展示预设引用方式

### 3. 典型规则表场景

- 固定附加费表
- 运费表
- 佣金表
- 类目 + 广告类型费率表
- 重量 + 折后售价联合查表

这个模块解决的问题是：

- 平台规则不能写死在页面里
- 规则应该可配置、可维护、可同步
- 创建页应该“查表计算”，而不是“硬编码计算”

## 四、创建页商品测算

创建页是整个项目的核心操作页。

它把“预设、选项、模板规则”真正串起来，用于录入商品并计算结果。

### 1. 创建页主要输入内容

- 预设组合
- 国家
- 平台
- 成本
- 重量
- 名称
- 款号
- 类目
- 广告类型
- 是否包邮
- 图片
- 变体
- 备注

### 2. 创建页主要计算内容

页面会围绕这些值做测算：

- 折前价
- 折后价
- 收入
- 总费用
- 净利润
- 利润率
- 活动费
- 交易费
- 提现费
- 汇损金额
- 税费
- 贴单费
- 卖家运费
- 固定附加费

### 3. 反推能力

创建页不是只有单向计算。

它支持把不同字段作为当前基准来反推结果，例如：

- 按折前价推
- 按折后价推
- 按收入推
- 按利润率推
- 按净利润推

### 4. 运算过程展示

创建页可以查看运算过程。

这部分会把当前计算拆成多个步骤展示，例如：

- 折扣系数
- 卖家运费查表
- 固定成本底数
- 综合费率
- 折前价 / 折后价关系
- 各费用项拆解
- 总费用
- 净利润
- 利润率

这个模块解决的问题是：

- 不只是给结果，还要让用户知道结果怎么算出来
- 便于检查公式、规则表、参数是否正确

### 5. 图片与变体

创建页还支持商品资料录入：

- 上传图片
- 管理多张图片
- 图片预览
- 图片和变体关联
- 维护多个变体组
- 每个变体组维护多个选项

这说明它不是纯利润计算器，而是“商品录入 + 计算”一体化页面。

### 6. 草稿与重置

创建页支持：

- 页面缓存
- 跳转其他页面再回来不丢
- 重置页面
- 重置后恢复到当前预设默认值

### 7. 添加到列表

创建页可以把当前商品生成正式记录并加入列表。

这是从“测算阶段”进入“沉淀阶段”的关键动作。

## 五、列表页商品沉淀与 Excel 同步

列表页是商品记录管理页。

它接收来自创建页的结果，并负责和另一份列表 Excel 同步。

### 1. 列表页主要能力

- 绑定列表 Excel
- 显示商品列表
- 展示商品图片
- 图片预览
- 删除商品记录
- 保存到 Excel
- 从已绑定 Excel 重新刷新

### 2. 列表中会保留什么

每条商品记录通常会保留：

- 名称
- 款号
- 国家 / 平台
- 类目
- 广告类型
- 是否包邮
- 当前基准
- 成本
- 重量
- 折前价
- 折后价
- 收入
- 利润率
- 净利润
- 各项费用
- 图片关系
- 变体关系
- 扩展字段
- 预设快照
- 计算快照

### 3. 为什么单独一个列表 Excel

因为商品记录和规则配置是两套完全不同的数据：

- 规则 Excel 更像配置中心
- 列表 Excel 更像结果库 / 商品库

分开后更清晰，也更安全。

这个模块解决的问题是：

- 商品测算结果不能停留在页面里
- 需要沉淀成可查看、可保存、可导回 Excel 的记录
- 图片和变体关系也要一起保留

## 六、Excel 驱动能力

这个项目的一个核心特征是“强 Excel 驱动”。

也就是说，Excel 不是最后导出一下，而是整个系统的重要数据源。

### 1. Excel 在项目里的角色

规则 Excel 负责：

- 预设
- 选项
- 模板规则

列表 Excel 负责：

- 商品记录
- 图片关系
- 变体关系
- 扩展字段

### 2. Excel 驱动带来的能力

- 可视化维护规则
- 不改代码就能调参数
- 业务侧可直接改表
- 程序内读取后可继续加工
- 修改后还能同步回原文件

### 3. 当前项目不是“只读 Excel”

它同时支持：

- 从 Excel 读取
- 在程序里编辑
- 再写回 Excel

所以本质上更接近：

- Excel 驱动的后台工具

## 七、图片能力

项目当前已经具备基本图片处理能力。

包括：

- 创建页上传图片
- 程序保存图片路径
- 创建页图片预览
- 列表页图片预览
- 列表 Excel 导出时尝试写入封面图
- 图片与商品、变体关系一起保存

它的意义在于：

- 商品记录不是只有数字
- 需要把图片也纳入记录体系

## 八、项目的整体数据流

当前项目的数据流可以概括成：

1. 规则 Excel 提供预设、选项、模板
2. 用户在创建页选择预设并录入商品
3. 创建页结合模板规则完成计算
4. 用户把当前商品加入列表
5. 列表页管理这些商品记录
6. 列表页把结果同步到列表 Excel

也可以简化为一句话：

- `规则配置 -> 创建测算 -> 列表沉淀 -> Excel 保存`

## 九、如果你要重写新项目，建议保留的功能骨架

如果只是提取功能，不考虑当前实现细节，我建议新项目至少保留这几层：

### 1. 配置层

- 预设
- 选项
- 模板规则

### 2. 计算层

- 创建页录入
- 结果计算
- 运算过程展示

### 3. 记录层

- 列表页
- 商品记录
- 图片
- 变体
- Excel 持久化

### 4. 文件层

- 规则 Excel
- 列表 Excel
